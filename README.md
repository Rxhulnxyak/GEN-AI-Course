## 📖 Overview
Customer service is one of the most critical aspects of any modern business. This repository documents the training phase of building a **Generative AI-powered Customer Service Bot**, capable of understanding customer questions, retrieving relevant information, and generating human-like responses in real-time. 

This project simulates a real-world enterprise customer support system commonly used by **e-commerce, SaaS, banking, and service-based companies**.

By completing this rigorous training curriculum, we gained hands-on experience in:
- 🧠 **Generative AI & Large Language Models (LLMs)**
- 🗣️ **Natural Language Processing (NLP)**
- 🔍 **Knowledge Retrieval (RAG & Semantic Search)**
- ⚙️ **Prompt Engineering**
- 💬 **Enterprise Chatbot Development**

## 📂 Repository Structure & Key Implementations

The training modules have been structured into three core components, each focusing on specific AI domains. 

### 1. `NLP` (Natural Language Processing)
The foundation of text processing and machine learning workflows.
- **`NLP_bot/`**: A foundational intent-based chatbot with custom GUI.
- **`NLP (1).ipynb` & `NLP_Word2vec.ipynb`**: Detailed notebooks covering TF-IDF implementation, tokenization, and Word2vec dense embeddings for contextual word representations.
- **Text Classification**: SMS Spam classification showing practical classical NLP tasks.

### 2. `LLM` (Large Language Models)
Deep dive into modern foundational models and integrations.
- **`End-To-End-Gemini-Project-main/`**: Complete projects leveraging the powerful Gemini AI.
- **`intro_palm_api.ipynb`**: Implementation of the Google PaLM API for automated text generation and reasoning.
- **`llama3/`**: Explorations into running state-of-the-art open-source LLMs locally.

### 3. `customer_service_chatbot_LLM` (Capstone Application)
The culmination of the training phase: a scalable, professional-grade Gen AI chatbot.
- **`src/`**: The core source code featuring advanced prompt engineering and API logic.
- **`dataset/`**: The vector stores and knowledge bases used for Retrieval-Augmented Generation (RAG).
- **Architecture**: Designed for real-time inference, high accuracy in domain-specific QA, and robust session handling.
